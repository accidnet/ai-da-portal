# AGENTS.md

- 기본적으로 언어는 한국어를 사용

## 저장소 개요

- `portal-web`: Vue 3 + TypeScript + Vite 기반 프론트엔드
- `portal-onl`: FastAPI기반 Python 백엔드

## 개발 가이드라인

- 구현에 바로 들어가기 전에 먼저 코드베이스와 변경 범위를 확인
- 동시에 작업이 가능한 경우에는 병렬로 처리할 것
  - 프론트, 백엔드의 작업 영역이 서로 영향이 없을 경우에도 병렬로 처리
- 보편적인 프로젝트 구조를 생각하며 폴더 및 파일을 생성할 것
- 변수, 상수, 함수, 클래스 등 외부, 내부 호출을 고려하여 구현
- 현재 `tests/`는 비활성화된 상태이므로 사용하지않음
- 개발 서버는 사용자가 직접 실행해 확인하는 경우가 많으므로, 명시 요청이 없으면 임의로 서버를 실행하지 말 것
  - 동작 확인이 필요하면 먼저 사용자가 이미 띄운 서버 기준으로 확인하고, 서버 실행이 꼭 필요할 때만 사용자에게 확인할 것

### `portal-web/src`

- Vue 3, TypeScript, Vite 기반 프론트엔드
- import 경로는 같은 depth의 모듈을 사용할 때만 상대경로(`.`)를 활용하고, 그 외 경로는 `@` 별칭을 활용할 것

### `portal-onl/src`

- 모든 실행에는 `portal-onl/.venv` 가상환경을 활성화 후 실행할 것
- `uv add`로 라이브러리 설치
  - 단, `torch`관련 라이브러리는 `uv pip install`로 할 것
- 기본적으로는 \_를 붙여 private을 나타내며, 확장, 외부호출, 공통의 가능성이 있을 경우에는 \_를 제거하여 개발
  - 의미없이 과도하게 코드를 쪼개서 많은 함수를 만들도록 하지 않음
- 같은 depth의 모듈을 사용할 경우에는 상대경로(.)를 활용하고, 그 외는 명시할 것
- 파이썬 프로젝트 구조는 `Feature-based` 구조를 따릅니다.
  - 초기에는 계층 구조로 설정되었으며, 현재는 코드 수정 및 추가될 때 점진적으로 전환을 하도록 합니다.

#### 백엔드 계층별 모델 배치 기준

- `api/**/schemas.py`: FastAPI 요청/응답 전용 Pydantic 모델을 배치
- `application/**/dto.py`: usecase, service, infra 사이에서 주고받는 데이터 구조를 배치
- `domain/**/models.py`: 도메인 자체의 핵심 개념과 규칙을 배치
- 새 모델을 추가하거나 기존 모델을 이동할 때는 위 책임을 기준으로 import 경로를 함께 정리할 것

#### 백엔드 DB 스키마 문서

- DB 테이블을 변경할 때는 `portal-onl/docs/database-schema.md`도 함께 현행화할 것

## 주석 및 docstring 작성 기준

- 코드를 작성하거나 수정할 때는 반드시 간략한 주석을 추가
- 작업한 코드 내 클래스 및 함수에 docstring이 작성되어있지않다면 추가하도록 함
- 리팩터링이나 파일 이동 시 기존 주석은 최대한 유지하고, 동작과 맞지 않는 경우에만 수정 또는 제거
