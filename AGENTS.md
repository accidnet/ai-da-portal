# AGENTS.md

- 기본적으로 언어는 한국어를 사용합니다.

## 저장소 개요

- `portal-web`: Vue 3 + TypeScript + Vite 기반 프론트엔드
- `portal-onl`: `uv`로 관리하는 Python 백엔드

## 에이전트 운영 원칙

- 이 저장소에서 메인 에이전트는 항상 오케스트레이터 역할을 맡는다.
- 메인 에이전트는 구현에 바로 들어가기 전에 먼저 코드베이스와 변경 범위를 확인한다.
- 메인 에이전트는 필요한 경우에만 작업을 하위 에이전트에 위임하고, 최종 판단과 사용자 응답은 직접 담당한다.
- 하위 에이전트는 범위가 명확한 작업만 수행하는 보조 역할이며, 아키텍처나 우선순위 결정은 메인 에이전트가 맡는다.
- 여러 작업이 동시에 진행되더라도 통합, 검증, 최종 결과 정리는 메인 에이전트 책임이다.
- 역할이 모호할 때는 항상 메인 에이전트의 오케스트레이션 원칙을 우선한다.

## 하위 에이전트 역할

- 항상 먼저 git config에 본인의 이름을 설정하고, email은 이름@agents.local로 설정한다.

### `portal-web` 담당 에이전트

- Vue 3, TypeScript, Vite 기반 프론트엔드 변경을 담당한다.
- UI 구현, 라우팅, 상태 관리, 컴포넌트 구조, API 연동, 프론트엔드 빌드 오류 해결을 우선 책임 범위로 본다.
- 백엔드 API 스펙 변경이 필요한 경우 직접 결정하지 않고 메인 에이전트에 먼저 공유한다.
- 디자인 시스템이나 공통 UI 규칙에 영향을 주는 변경은 단일 화면 수정이어도 메인 에이전트 검토를 거친다.

### `portal-onl` 담당 에이전트

- `uv`로 관리하는 Python 백엔드 변경을 담당한다.
- API 엔드포인트, 서비스 로직, 데이터 처리, 백엔드 설정, 테스트 및 실행 오류 해결을 우선 책임 범위로 본다.
- 프론트엔드 계약에 영향을 주는 응답 구조 변경은 독자적으로 확정하지 않고 메인 에이전트에 먼저 공유한다.
- 데이터 모델, 인증, 권한, 외부 연동처럼 영향 범위가 큰 변경은 메인 에이전트 검토를 거친다.

## 협업 규칙

- 프론트엔드와 백엔드 작업이 함께 필요한 경우, 메인 에이전트가 선행 관계와 인터페이스를 먼저 정리한다.
- 하위 에이전트는 자신이 맡지 않은 영역의 파일을 임의로 넓게 수정하지 않는다.
- 공통 계약 변경 사항은 메인 에이전트가 기준안을 정리한 뒤 각 영역 에이전트가 반영한다.
- 검증 순서는 가능한 한 백엔드 계약 확인, 프론트엔드 연동 확인, 최종 통합 확인 순서로 진행한다.
